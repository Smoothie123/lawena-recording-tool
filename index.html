What's this?

Lawena is a simple Java 7+ tool which greatly enhances your TF2 and other Source games image quality for recording purposes, with surprisingly little performance loss compared to other such configurations. Records using ingame Source Recorder, with key bindings to ease the process.

Easy-to-use and no install required. This tool does not interfere with your regular TF2 configuration, HUD or launch options. Requires Steam in order to run.

Features

  • Insane max quality graphics configuration.
  • Recorded files are saved to the directory of your choosing for comfort and order.
  • Options to enable/disable various game elements which may be unnecessary in the context of a movie.
  • Includes built-in moviehuds with different levels of simplicity.
  • Supports custom HUDs/materials/models/etc to add a personal touch to your movies.
  • Includes a set of built-in skyboxes, and supports custom skyboxes via simple drag-and-drop.
  • SrcDemo2 support to reduce drive usage and space.
  • VDM file support for automatizing your movie recording process.
  • Updates automatically, you can see the latest changes here.

Installing Lawena

If you want to use a stable version, check the releases page. Simply download the zip release, extract it anywhere and run lawena.exe.

Known Issues

  • If you have your own class configs with bind command in them, you can lose some of the lawena's own keybindings, because class configs get recorded into the .dem files. Just check out recbindings.cfg and execute the equivalent console command.
  • Selecting your own custom/mystuff/cfg files to load into TF2 can cause loss of recording quality and lawena's keybindings.
  • Some keybindings might be missing, an alternative is to type startrecording into the console, otherwise check out you're not playing a demo that rebinds some keys and if it does, edit recbindings.cfg inside lawena cfg folder accordingly to solve it.
  • Custom HUDs can trigger a "Failed to restore files" due to Windows locking a custom font file. In case your custom files somehow get misplaced and disappear upon restoring, there are backup archives of your custom files in your tf folder named lawena-user.timestamp.bak.zip.
  • Choosing a movie/recording folder with special characters like brackets or accents might confuse Source Recorder and not save any frames. Use simpler path names.

Latest development builds

There's some pre-releases that preview upcoming content. To download the latest, not particularly stable version:

  • Download this repository and extract it to any folder.
  • Run gradlew.bat (Windows) or gradlew script (Unix) to start the program. Administrative permissions are required due to the use of symbolic links.

You can also switch updater branches from within the tool, just go to the Help -> Switch Updater Branch... menu item, choose one of the branches, press OK and restart the tool to apply the changes. The current branches are:

  • release-v4.1 — latest stable version (Recommended)
  • preview-v4.1 — latest updates being tested
  • release-v4.2/snapshot-v4.2 — newer version featuring symbolic links, multiple Source game support and other features. (Unstable!)
